# 一肢入魂 デザインブリーフ(Claude Design 用)

Claude Design にアップロードする資料と、チャットに貼るプロンプトのセット。

---

## 渡すファイル一式

1. **takken-zenshi-game.jsx** — 実装済みプロトタイプ。デザインシステムの現物(色・フォント・コンポーネント)はすべてここにある
2. **takken-game-design-v1.md** — 設計ドキュメント。画面構成と今後追加するエンジン(計算・間違い探し)の仕様
3. **このブリーフ** — 下のプロンプトをチャットに貼る
4. (あれば)現行画面のスクリーンショット 3枚(範囲選択/プレイ/判決)

---

## チャットに貼るプロンプト(初回・コピペ用)

```
宅建(宅地建物取引士試験)の学習ゲーム「一肢入魂」のUIデザインを一緒に作りたい。
添付のjsxが実装済みプロトタイプで、この中のデザインシステムを正として維持してほしい。

## プロダクトの核
- 4択を当てるゲームではなく「全部の肢に理由をつけて決着をつける」ゲーム
- 判定→誤り箇所タップ→理由選択の3段階。勘の正解では点が伸びない
- ユーザーは通勤中などにスマホで遊ぶ社会人。1セッション5〜10分

## デザインシステム(jsxから抽出して使うこと)
- 世界観: 法律文書 × 朱印。「開廷する」「判決」「合格/追試のスタンプ」という司法モチーフ
- 紙: #ECEEE9 / カード: #FBFAF7 / 墨(テキスト): #26333B
- 藍(インタラクティブ): #33557E / 朱(判定・強調): #BF3B33 / 緑(正解): #3D7A55
- 見出し: 明朝(Hiragino Mincho ProN系) / 本文: ゴシック(Hiragino Kaku Gothic系)
- 角丸10px、罫線 #D8DAD2、判定は朱印スタンプが回転しながら押される演出
- 法律用語には青点線の下線、タップで画面下に用語辞書ポップアップ

## 制約
- モバイルファースト(コンテンツ幅 max 560px)。タップ領域は44px以上
- Web フォント不使用(日本語システムフォントのみ)
- 最終的に Next.js + React で実装する。装飾はCSSで再現可能な範囲で
- ダークモードは今回対象外

## トーン
- 好き: 逆転裁判の緊張感、無印良品的な余白、判子・和紙などの和の意匠
- 嫌い: ゲーミフィケーション過剰(コイン・宝箱・レベルアップ演出)、
  グラデーション多用、絵文字だらけのUI、子供っぽいマスコット

まず、添付資料を読んだ上でデザインシステムの理解を要約して見せて。
その後、依頼する画面を順に伝える。
```

## 依頼したい画面(優先順)

初回プロンプトの後、1画面ずつ依頼する。まとめて頼むと品質が下がるので分割推奨。

### ① 報酬計算エンジンの解答画面(未実装・最優先)
```
設計mdの type: "calc" 用の画面。宅建業法の報酬計算問題
(例: 売買価格3,000万円の媒介報酬の上限額を求める)を解く画面を3案。
要件:
- 与件(価格・取引態様・消費税)がカード上部に整理されて見える
- 数値入力 or 選択式の解答UI(スマホで打ちやすいこと)
- 正解後に途中式がステップ表示される領域
- 既存の「肢カード+朱印スタンプ」の文法から逸脱しないこと
```

### ② マイソク間違い探し画面(未実装)
```
設計mdの type: "spot" 用。不動産広告(マイソク)を模したカードを表示し、
広告規制違反の箇所をタップで探すゲーム画面を3案。
要件:
- 広告部分は「本物っぽさ」重視(物件写真枠・間取り・徒歩分数・価格表記)
- 発見済み/未発見の箇所数表示(例: 2/4)
- タップした箇所に朱の付箋やマーカーが付く演出
- 広告の「それっぽさ」とゲームUIの和風世界観の共存が課題。案ごとに解き方を変えて
```

### ③ 成績ダッシュボード(v1.5用)
```
弱点復習・間隔反復の起点になる成績画面。
分野×論点のマトリクスで習熟度が一望でき、「今日やるべき肢」が提示される。
朱印(完璧)/藍(学習中)/白(未着手)の3状態で塗り分けるイメージ。1〜2案。
```

### ④ 既存3画面のブラッシュアップ(最後)
```
範囲選択/プレイ/判決の既存画面について、構造は変えずに
余白・タイポグラフィ・階層の微調整だけ提案して。Before/After比較で。
```

## 反復時に使う指示の例

- 「案2の方向で。ただし広告部分の情報密度をもっと上げて、実際のマイソクに寄せて」
- 「スタンプ演出はそのまま、色数を1色減らせるか試して」
- 「この画面をClaude Codeに引き継ぐ前提で、コンポーネント分割の境界にコメントを入れて」

## 終わったら

固まった案は Claude Code に引き継ぎ(既存成果物を引き継げる)、
Next.js プロジェクトの components/engine/ 配下に実装する。
