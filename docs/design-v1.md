### 追加ジャンルの優先順位(合格戦略ベース)

## 2. データ設計

### 2.1 スキーマ(TypeScript)

```typescript
// 分類は3層: category > topic > question
type Category = "権利関係" | "宅建業法" | "法令上の制限" | "税・その他";

type Question = {
  id: string;              // "minpo-nijujoto-001"
  category: Category;
  topic: string;           // "二重譲渡"
  law: string;             // "民法177条"
  type: "zenshi" | "calc" | "spot";  // 全肢判定 / 計算 / 間違い探し
  scenario?: string;       // 事案文(業法の知識問題では省略可)
  lesson: string[];        // 30秒レッスン
  diagram?: Diagram;       // 関係図(省略可)
  choices: Choice[];       // type="zenshi" のとき
  calc?: CalcSpec;         // type="calc" のとき
  spot?: SpotSpec;         // type="spot" のとき
  tags?: string[];         // "個数問題頻出" "改正点" など
  sourceNote?: string;     // 元ネタ過去問の年度メモ(内部用)
};

type Choice = {
  segments: string[];      // 誤り箇所タップ用に分割した文
  correct: boolean;
  wrongIndex?: number;     // 誤り肢のみ
  reasons: Reason[];       // ○肢=根拠 / ×肢=誤りの理由
  explanation: string;
};

// 計算形式(報酬計算・建蔽率容積率・相続分)
type CalcSpec = {
  prompt: string;
  given: { label: string; value: string }[];  // 与件の整理表示
  answer: number;
  unit: string;            // "円" "㎡" など
  tolerance?: number;
  steps: { label: string; formula: string }[]; // 解説用の途中式
  distractors: number[];   // よくある計算ミスの値 → 選択式にも流用可
};

// 間違い探し形式(マイソク/広告)
type SpotSpec = {
  layout: AdElement[];     // HTML/CSSで組む広告の要素ツリー
  errors: { elementId: string; reason: string; law: string }[];
  errorCount: number;      // 「N箇所ある」と提示
};
```

ポイント:
- **`type` でエンジンを分岐**。現行の全肢判定は `zenshi`。計算と間違い探しは後から追加してもデータ構造が壊れない
- **`diagram` と `scenario` はオプショナル**(実装済み)。業法の手続き問題は図なしでよい
- `sourceNote` で「どの年度の過去問を参考にしたか」を内部管理。肢の文言は自作(オリジナル)で書く

### 2.2 用語辞書


### v3(Neon移行)の前提条件 — 当面見送り

当面はDBを分けず、localStorage を継続する(Issue #50 で決定)。

理由:

- `StorageAdapter` の抽象化(3.2節)により、Neon移行は将来 Adapter 1ファイルの差し替えで済む。前倒しする利益がない
- DB化は認証・サーバー・運用・費用を連れてくる。単一端末・個人利用の現状には過剰
- localStorage の唯一の実リスク(データ消失)は、バックアップ機能(エクスポート/インポート。実装済み)で対応済み

以下の**いずれか**を満たしたら v3 に着手する:

- 複数端末で成績を同期したくなった
- アプリを公開し、他ユーザー/ランキングが必要になった
- localStorage の容量超過・破損が現実に発生した

それまでは、Adapter 差し替えで壊れないこと(アプリ側が localStorage を直接触らないこと)を保つことだけ意識する。

### 問題作成の運用メモ

- 肢の文言は過去問の「論点」だけ借りて自作する(表現のオリジナリティ確保と、segments 分割・理由3択に最適化するため)
- 1テーマ = 事案1つ + 4肢 + レッスン3行 + 用語 が制作単位。慣れれば1テーマ30〜60分
- 法改正チェック: 民法(2020大改正)、盛土規制法(2023)、クーリングオフ告知等の電磁的方法の扱いは要注意ゾーン
